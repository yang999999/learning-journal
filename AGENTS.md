# AGENTS.md — learning-journal 项目规则

## 🎯 项目目标
面向**后端工程师转型 AI Agent 方向**的体系化学习仓库，目标是在面试中后端 + AI Agent 知识满分，同时用真实业务（直播电商C端/直播间召回/用户行为存储）包装 2 个 AI 项目应对面试深挖。

## 📂 仓库结构
- `docs/resources/` — 后端 & AI 面试知识点速查文档（推送 GitHub）
- `projects/` — 简历项目 A/B 包装文档
- `weekly/` — 周计划
- `scripts/` — 工具脚本（如 quiz.sh 抽题检验）
- `CURRENT_PROGRESS.md` — **当前学习进度（必读）**，每次会话开始读它了解进展
- `AGENTS.md` — 本文件（每次新会话自动加载）

## 🔄 会话规则（重要！）
1. **每次会话开始时**：必须先读 `CURRENT_PROGRESS.md` 了解当前进度，然后再响应用户。
2. **每次完成任务后**：必须更新 `CURRENT_PROGRESS.md`，把已完成内容标记 ✅，下一个任务更新到"待学习"。
3. **推送 GitHub 必须验证**：推送后要 `git log origin/main -1` 确认 commit 真的在远端，再告诉用户完成；失败必须主动告知。
4. **任务未完成必须报告**：如果任务被限流/中断/错误打断，**下一次回复开头必须用 `⚠️ 任务状态报告` 格式说明**哪个任务没完成、原因、进度。
5. **不要一次做太多任务**：每次只做 1-2 个任务，做完更新进度文件再继续，避免上下文爆炸。

## 💡 学习模式约定
- 学习流程：**知识点学习 → 答案整理到 docs/ → 抽题检验 → 通过后标记完成 → 进入下一个**
- 后端部分不需要实战（工作中已经在用），重点是面试题 + 技术深挖
- AI Agent 部分需要：知识点 + 面试题 + 项目包装（项目细节要能应对深挖，不要求完整跑通）
- 每日学习步骤：根据 `CURRENT_PROGRESS.md` 的"下一个待学习任务"开始
- 完成后推送 GitHub，并更新进度文件

## 🚫 禁止行为
- 不要一次 cat 整个大文件，用 grep/head/精准定位读取
- 不要输出一堆 "done"，完成就明确汇报
- 不要在没有验证推送成功的情况下声称"已经推送"

# learning-journal
目标：后端（Go）+ AI Agent 开发面试满分。

## 核心原则（最重要）
- **面向面试满分**：所有知识点、项目包装终极目标是面试中答满分、经得住深挖细节。
- **项目优先讲清楚**：项目要能讲得像自己真实做过，细节（架构、难点、踩坑、决策、数据）必须扛住3轮深挖。动手demo可选，重点是理解到"自己做过"的深度。
- **技术架构真实合理**：不强行全Go。主流大厂AI落地架构——Go扛高并发网关/业务接入层，Python跑AI Agent编排层（LangGraph/RAG/模型调用），gRPC通信。
- **学技术的应用和应对面试**，不纠结实现细节但要懂关键trade-off。

## 真实业务背景（AI项目素材来源）
方向：**TikTok 直播电商 C 端流量承接**
负责业务：商品卡/购物袋/预览流/秒杀玩法/竞拍/直播间召回（GMV/观看人数维度）/用户行为存储（观看时长/购买/下单/点击）。
包装AI项目从这些真实业务里挑，必须真实可信。

## 项目统一方向
**只保留一个项目**：《直播电商智能助手》（见 `projects/直播电商智能助手.md`）
- 场景1：**内部OnCall+运营取数**——输入region/pid/时间范围/活动ID等零散信息，自动补参追问→并行查Prometheus/Loki/发布/CMDB/用户行为→汇总根因建议→高危操作HITL人工审批
- 场景2：**C端智能客服**（接入已有客服体系）——直播间用户问价/秒杀/竞拍/规则，数据脱敏，三层防幻觉，高并发支撑秒杀峰值
- 两个场景共享一套AI底座（模型路由/RAG/Guardrail/工具双模式/缓存/可观测），通过入口+权限+工具白名单+Prompt隔离

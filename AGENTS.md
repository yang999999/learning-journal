# learning-journal
目标：后端+AI Agent 开发面试满分。

## 核心原则（最重要）
- **面向面试满分**：所有知识点、项目包装的终极目标是面试中能答满分、经得起深挖细节。
- **项目部分优先讲清楚**：项目要能给面试官讲得像自己真实做过，细节（架构、难点、踩坑、决策、数据）必须能扛住3轮深挖。
- **动手写代码可选**：有时间可以写demo，没时间优先把技术原理+项目设计+落地细节理解到"自己动手写过"的深度。
- 用直播电商业务（直播间召回/用户行为）包装2个AI项目，必须真实可信。

## 必读
- 会话开始先读 `CURRENT_PROGRESS.md` 了解进度；`docs/LEARNING_LOG.md` 非必要不读

## 规则
- 每次完成任务后更新 `CURRENT_PROGRESS.md`，保持其 ≤80 行，旧项归档到 `LEARNING_LOG.md`
- 推送后用 `git log origin/main -1` 验证成功再收尾
- 任务被打断时下次回复开头用 `⚠️ 任务状态报告` 说明
- 每次只做1-2个任务，不一次堆太多
- 大文件用 grep/head 精准读，不 cat 整文件
- 知识点写进 `docs/resources/`，不写进进度文件

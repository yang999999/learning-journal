# learning-journal
目标：后端（Go）+AI Agent 开发面试满分。

## 核心原则（最重要）
- **面向面试满分**：所有知识点、项目包装终极目标是面试中答满分、经得住深挖细节。
- **项目部分优先讲清楚**：项目要能讲得像自己真实做过，细节（架构、难点、踩坑、决策、数据）必须扛住3轮深挖。
- **动手写代码可选**：有时间写demo，没时间优先把技术原理+项目设计+落地细节理解到"自己动手写过"的深度。
- **技术架构真实合理**：不强行全Go，采用主流微服务架构——Go负责高并发网关/业务接入层，Python负责AI Agent编排层（大模型调用/LangChain/LangGraph更成熟），两者通过gRPC通信。

## 真实业务背景（包装AI项目的素材来源）
方向：**TikTok 直播电商 C 端流量承接**
主要负责业务：
- 商品卡展示（营销价格、秒杀信息、商卡排序）
- 购物袋功能
- 预览流
- 秒杀玩法
- 竞拍玩法全链路
- 直播间召回（按GMV/观看人数等维度）
- 用户行为存储（观看时长、购买、下单、点击等）
包装AI项目时从这些真实业务里挑场景，必须真实可信，不要局限于召回/行为存储两点。

## 必读
- 会话开始先读 `CURRENT_PROGRESS.md` 了解进度；`docs/LEARNING_LOG.md` 非必要不读

## 规则
- 每次完成任务后更新 `CURRENT_PROGRESS.md`，保持其 ≤80 行，旧项归档到 `LEARNING_LOG.md`
- 推送后用 `git log origin/main -1` 验证成功再收尾
- 任务被打断时下次回复开头用 `⚠️ 任务状态报告` 说明
- 每次只做1-2个任务，不一次堆太多
- 大文件用 grep/head 精准读，不 cat 整文件
- 知识点写进 `docs/resources/`，不写进进度文件

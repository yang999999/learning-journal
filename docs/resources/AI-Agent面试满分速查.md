## 一、大模型基础（必问，面试第一个问题区）

### Q1：解释一下 Self-Attention？为什么要除以 √d_k？
**满分答案框架**：
> **核心本质**：解决RNN/LSTM串行不能并行问题，让所有词同时计算，GPU利用率更高，能捕捉长距离依赖。
>
> **数学公式：** `Attention(Q,K,V) = softmax(QK^T/√d_k) · V`
>
> 每个词通过三个线性投影得到三个向量：
> - Q (Query)："我要找什么信息，我查询谁"
> - K (Key)："我有什么信息，别人可以来查我"
> - V (Value)："我的信息是什么"
>
> Q 和所有 K 做点积得到注意力分数（就是相似度，越相似分数越高）。
>
> **为什么除以 √d_k？**
> 假设 Q 和 K 每个元素是均值0方差1，点积方差就是 `d_k`（sum 乘积方差就是d_k倍单个方差）。除以 √d_k 后方差回到1，避免softmax分数太极端（大部分概率堆到少数几个点），保持梯度稳定，让模型容易训练。
>
> **Multi-Head Attention（多头）**：
> - 把Q/K/V切分成h个小头，每个头维度是 `d_k = d_model / h`，每个头单独算一遍Self-Attention
> - 最后把所有头输出拼起来，乘一个投影矩阵得到最终输出
> - 为什么要多头：不同的头能学到不同的注意力模式，同一个词能同时关注句子里不同位置的不同语义关系（比如指代消解，一个头关注先行词位置，一个头关注动词关系），表达能力更强。
>
> 最后把所有头输出拼起来乘以投影矩阵，得到和输入维度一样的最终输出，总参数量没变，只是多了一次切分和拼接，代价很小，提升明显。

**手算例子（3个词、d=2，完整展示流程）**

以句子"我 喜欢 你"为例，维度d_k=2，Wq/Wk/Wv简化为2×2矩阵：

1. **每个词初始Embedding**：x_我=[1,0]，x_喜欢=[0,1]，x_你=[1,1]
2. **算Q/K/V**：每个词x乘Wq/Wk/Wv得到Q/K/V，例如Q_我=x·Wq=[1,0]，K_我=[0,1]，V_我=[1,0]
3. **点积算分数**：每个词的Q和所有词的K点积（对应元素相乘相加），Q·K越大表示越相关
   - "我"的分数 = [Q_我·K_我=0, Q_我·K_喜欢=1, Q_我·K_你=1]
4. **Softmax归一化**：分数变权重，加起来=1
   - "我"的权重 = softmax([0,1,1]) ≈ [0.16, 0.42, 0.42] → 表示"我"融合信息时42%来自"喜欢"、42%来自"你"
5. **加权求和**：输出 = 0.16×V_我 + 0.42×V_喜欢 + 0.42×V_你 = [1.0, 0.84]
   - 原始"我"是[1,0]，新表示融合了"喜欢"和"你"的V向量信息，第二个维度从0变成0.84，就是吸收了上下文

**白话理解**：
- Q是"我要找谁"，K是"我的特征"，V是"我的信息"
- 点积Q·K = 匹配度（你和我像不像）
- Softmax = 匹配度转成百分比权重
- 加权求和 = 按权重吸收相关词的信息
- 最终每个词都变成融合了上下文的新向量，这就是"自注意力"理解上下文的原理

**实战对应**：如果你做过模型微调，说："我微调 LLaMA 时看过源码，Self-Attention 部分确实就是这么实现，我还对比过 RoPE 对精度影响"。

### 追问必背（100%会问）

**1. 完整 Self-Attention 计算流程（手算级别）**
以3个词d=2为例：
1. 输入 X 乘 W_Q/W_K/W_V 三个可学习矩阵，得到 Q/K/V 向量
2. 点积：score(i,j) = Q_i · K_j（相似度，Q是"我找什么"，K是"我有什么"）
3. 缩放：除以 √d_k，防止维度大点积过大Softmax饱和梯度消失
4. Softmax：每行 exp/Σexp，归一化成权重 α（每行和=1）
5. 加权求和：O_i = Σ_j α_ij · V_j，每个词的输出是所有词V的加权平均，权重视相关性

**2. 为什么多头（Multi-Head）？**
- 类比多路召回：单头只能学一种相似关系，分头并行学不同子空间（语法/语义/位置）
- h个头，每头 d_k = d_model/h，总计算量和单头相同
- 每个头独立有自己的 W_Q/W_K/W_V，算完 Concat 再乘 W_O 融合
- 工程上一次大矩阵乘法并行完成，不增延迟

**3. KV Cache（推理加速核心，必问）**
- **问题**：自回归生成第t个token时，前t-1个token的K/V重复计算
- **解法**：历史token的K/V是不变的（因为输入已定），prefill阶段把所有prompt的K/V算好缓存，decode阶段每步只算新token的K/V并append
- **加速效果**：单步decode从O(n²)→O(n)，整体O(n³)→O(n²)
- **显存代价**：KV_cache = 2×n_layers×n_kv_heads×d_head×seq_len×batch×2字节
  例：Llama2-7B seq=4096 batch=1 ≈ 2GB；batch=32 ≈ 64GB，是显存主要瓶颈

**4. MHA vs MQA vs GQA（省显存设计）**
| 模式 | Q头 | KV头 | 代表 | KV显存 | 效果 |
|---|---|---|---|---|---|
| MHA | h | h | BERT/GPT-3 | 1x | 最好 |
| MQA | h | 1 | PaLM | 1/h 最省 | 掉点明显 |
| GQA | h | h/k 分组 | **Llama2-70B(32Q/8KV)**、Qwen、Mistral | 1/k 折中 | **几乎不掉点** ✅ |
- GQA是当前大模型推理优化主流：KV Cache显存降k倍，吞吐提升2-3倍，效果损失极小

**5. RoPE旋转位置编码**
- 老办法（BERT/GPT-2）：可学习绝对位置embedding，不能外推长度
- RoPE：对Q/K每两维看成2D点，按位置m旋转θ角，旋转后点积只依赖**相对位置差**
- 优势：天然相对位置感知 + 支持长度外推（NTK/YaRN插值，4K训的模型扩到128K）
- 主流模型（Llama/Qwen/Mistral）全用RoPE

**面试金句**："Transformer推理成本大头是KV Cache显存，所以现代模型普遍用GQA省KV Cache，RoPE支持长上下文外推，推理框架如vLLM用PagedAttention解决KV显存碎片问题。"

### Q2：RAG 完整链路说一遍？每个环节怎么做优化？
**满分答案框架（按顺序说，有结构）**：

```
[索引阶段]
  文档加载 → 文档切片 → Embedding → 向量存储
[推理阶段]
  用户提问 → Query预处理 → 检索 → Rerank → Prompt拼接 → LLM生成 → 后处理 → 返回
```

**每个环节面试要点 + 实战点**：

| 环节 | 高频面试点 | 满分答法 | 简历写什么 |
|------|------------|----------|-----------|
| **切片 Chunking** | 怎么选 chunk size？为什么要 overlap？ | 固定大小 256~1024 tokens，overlap 10%~20%，防止切断语义。进阶：递归字符切片优先切自然边界（段落→句子），语义切片找相似度断点。生产常用父子文档（小块检索大块生成）。 | 项目中对比了 256/512/1024 三种 chunk size，在我司知识库上 512 最优，Recall@5 提升了 8% |
| **Embedding** | BGE-M3 为什么好用？ | BGE-M3 支持多语言、长文档、多粒度嵌入，国内开源第一梯队，效果比 OpenAI text-embedding-3 差一点但免费可部署。中文场景优先 BGE-M3/M3E。 | 项目中替换旧 embedding 为 BGE-M3，Recall@5 从 78%→86% |
| **检索** | 混合检索怎么做？ | 向量擅长语义匹配，BM25 擅长精确关键词匹配，两路召回后用 RRF（Reciprocal Rank Fusion）融合排名，不调权重泛用性好。 | 实现了混合检索，相比纯向量 Recall@5 又提升了 5% |
| **Rerank** | 为什么需要 Rerank？ | 多路召回选出 Top20 后，用 Cross-Encoder（把 query+doc 一起喂给模型）精排，计算量大但精度高，能大幅提升 precision@k。常用 bge-reranker-v2-m3。 | 项目中加了 Rerank，最终回答 Faithfulness 提升了 12% |
| **Query 优化** | 最常用的优化方法 | ①HyDE：先生成假设答案，用假设答案检索；②Multi-Query：生成多个角度查询并行检索；③Step-Back：先抽象问题再检索；④Query 改写：多轮对话做指代消解。 | 我在项目中集成了 HyDE + Step-Back，解决了问题语义模糊漏检问题 |
| **幻觉** | 怎么减少 RAG 幻觉？ | ①Prompt 强约束：只回答上下文里有内容；②强制引用来源，让模型标出来源段落；③Faithfulness 校验：生成后用 LLM-as-Judge 校验是否每个事实都来自上下文；④不相关上下文直接返回不知道。 | 实现了 LLM-as-Judge 自动检测幻觉，幻觉率从 18%降到 5% |

**追问必背**：
> Q：Graph RAG 是什么？解决什么问题？
> A：微软提出，先从文档抽实体/关系建知识图谱，然后基于图遍历回答全局问题（如总结全文观点），解决传统 RAG 对跨文档全局查询差的问题。缺点是抽取成本高、复杂知识图谱维护难。适合全局知识问答，不适合问答具体文档片段。
>
> Q：Agentic RAG 是什么？
> A：让 Agent 自主决策检索流程——判断问题需要检索吗？检索不对需要改写再检索吗？多轮检索直到拿到足够信息再回答。比传统固定一次检索效果好，但是增加延迟和成本。

**实战对应代码**：`projects/week1-quickstart/05_mini_rag.py` 就是完整的 RAG 最小实现，代码量 200 行，看懂了面试就能说清链路。

### Q3：LoRA 原理？为什么能有效？LoRA vs QLoRA vs 全参微调怎么选？
**满分答案框架**：
> 核心思想：预训练权重 W₀ 冻结，在旁边加低秩旁路 BA，前向 `h = W₀x + BAx`，只训练 A 和 B（r<<min(d,k)，参数少 100~1000 倍）。
>
> **为什么有效**：论文证明，下游任务的权重更新本质上在一个低秩子空间，低秩矩阵足够表达变化，不损失精度。训练初期 A 初始化为 0，B 高斯小方差，所以刚开始 BA=0，不影响原模型。
>
> **选型表**：
> | 方案 | 显存需求 | 效果 | 适用场景 |
> |------|----------|------|----------|
> | 全参 | 最高（7B fp16≈140GB） | 最好 | 大规模数据+多卡A100 |
> | LoRA | 中等（7B≈10-20GB） | 接近全参（95%+） | 绝大多数场景首选 |
> | QLoRA | 最低（7B≈5-8GB 可训，24GB可训70B） | 几乎和 LoRA 一致 | 消费级 GPU、大模型 |
>
> **推理**：可以 merge BA 到 W₀（零开销），也可以保留 adapter 插拔（多场景多 adapter 灵活）。

**追问必背**：
> Q：QLoRA 为什么 4bit 量化几乎不损失效果？
> A：用 NF4（Normalized Float 4bit）量化，适配预训练权重正态分布；双量化进一步压缩；训练时 paged optimizer 处理 OOM；量化误差被 LoRA 吸收，所以效果几乎和全精度 LoRA 一样。
>
> Q：为什么微调后效果反而掉了？可能有哪些原因？
> A：①数据质量差（低质/有毒/分布偏移大）；②学习率太大/太小；③过拟合（数据量小，epoch 太多）；④灾难性遗忘（没加通用数据）；⑤LoRA rank 选太大/太小；⑥数据格式不对（多轮对话格式错了）。

**实战对应项目**：W4/W5 做一个 QLoRA 微调垂直领域小模型，产出评测报告放简历上。

### Q4：DPO 和 RLHF 有什么区别？为什么 DPO 现在这么流行？
**满分答案框架**：
> RLHF 三阶段：SFT → 训练 Reward Model → PPO 优化。工程复杂，训练不稳定，需要单独训 RM，PPO 容易奖励黑客。
>
> DPO：直接偏好优化，跳过 RM，直接用「(chosen, rejected)」偏好对优化策略，数学上等价于 RLHF 最优解，训练简单稳定，不需要 RM，不需要 PPO。现在工业界做对齐都转 DPO 了。
>
> 如果你时间少，对齐直接走「SFT → DPO」两步就完了。

**追问**：ORPO 是什么？直接偏好优化在 SFT 阶段就做，更简单。

### Q5：Agent 和普通 ChatBot 本质区别是什么？Agent 核心组件？
**满分答案框架**：
- 普通 ChatBot：**只有文本生成**，输入问题 → 输出回答，不会用工具，不能感知环境，不能多步推理。
- Agent：**能自主行动完成任务**，核心五个组件：
  1. **大脑（LLM）**：做决策
  2. **规划**：分解任务、多步规划（CoT/ToT）
  3. **工具使用**：调用 API/数据库/搜索/code interpreter
  4. **记忆**：短期（上下文）/长期（向量库）/工作（scratchpad）
  5. **行动循环**：Thought → Action → Observation 循环直到完成（ReAct 范式）。

**追问必背**：
> Q：LangGraph 和传统 LangChain Agent 有什么区别？为什么现在 LangGraph 是主流？
> A：LangChain 传统 Agent 是黑盒循环，LMM 自己决定什么时候结束，不支持复杂分支、断点恢复、人工干预。LangGraph 用**显式状态机图**：节点是操作，边是状态转移，条件路由明确，内置 Checkpointer 自动持久化状态，支持中断/恢复/人工审批，多 Agent 协作清晰。我最近项目就是用 LangGraph 做 Supervisor-Worker 多 Agent，可控性强太多了。

### Q6：Agent 生产落地最大挑战是什么？怎么解决？
**满分答案框架（结构化答，分点说）**：

| 挑战 | 原因 | 解决方案 |
|------|------|----------|
| **非确定性** | LLM 可能格式错/幻觉/死循环 | 强制结构化输出+OutputParser 重试+最大步数限制+Guardrails |
| **延迟高** | 多步工具调用+长上下文+大模型推理 | 流式输出+并行工具调用+缓存（语义缓存+精确缓存）+模型路由（小模型答简单问题） |
| **成本高** | 多轮多步 token 累积 | 精简 Prompt+压缩上下文+模型路由+token 预算控制 |
| **可观测性差** | 多步执行出问题难排查 | LangSmith/Langfuse 全链路 Trace，每一步 prompt/工具结果/tokens 都记录 |
| **安全风险** | Prompt Injection → 让 Agent 执行危险操作 | 工具白名单+输入审查+敏感操作人工审批+数据脱敏 |
| **状态管理** | 长会话/多轮工具调用状态一致性 | LangGraph Checkpointer 持久化状态，thread_id 隔离 |

如果你是后端工程师，这里可以加分："我们后端做服务的思路很适合解决这些问题——高并发就是分层缓存、异步任务队列、熔断降级，和常规微服务一样的工程方法论套过来就行。"

### Q7：MCP（Model Context Protocol）是什么？为什么重要？
**满分答案框架**：
- MCP 是 Anthropic 提出的开放协议，统一大模型和外部工具/数据源的连接方式。
- 之前每个框架/每个工具都要定制适配，N 个 Client × M 个 Server 要 N×M 个适配。
- MCP 定义了 Tools/Resources/Prompts 三类原语，统一 Client-Server 架构（stdio/SSE 传输）。一个 MCP Server 可以被所有支持 MCP 的 Client（Claude Desktop/Cursor/Codex/LangGraph）直接使用，所以说像 USB-C 统一外设接口。
- 2026 年已经成为 Agent 工具接入的事实标准，我最近做的项目也接入了。

## 📝 AI Agent 面试答题总结

1. **先讲原理，再讲你做过的项目**：不要只说理论，一定要带自己做了什么，得到什么指标提升，面试官最爱听。
2. **后端优势放大**：讲部署/性能/可观测/安全的时候，把你后端经验套进去，这是你的差异化。
3. **最新动态要知道**：MCP、LangGraph、DPO、QLoRA 这些 2025-2026 主流技术要会说，说明你跟进前沿。
4. **不要硬编**：不会就说"这块我实践不多，但我的理解是...回去我再深入看一下"，诚实比瞎说强。### 🔹 基础概念三兄弟：Token / Temperature / Prompt（必背）

这三个是所有 AI 面试开场最基础的问题，必须1分钟讲清楚。

#### Token 是什么？
- **Token 是大模型处理文本的最小单位**，不是字，不是词，是一种通过 BPE（字节对编码）分词得到的"中间单位"
- 中文 1 字 ≈ 1.5-2 tokens，英文 1 单词 ≈ 1-1.5 tokens
- **为什么要关心 Token**：① 计费按 token 算（输入+输出分开）；② 上下文窗口限制（如 GPT-4o 128K ≈ 10万中文字）；③ 超长内容需要截断/摘要
- BPE 算法原理：先把文本拆成单字节，反复合并高频字节对，直到达到词表大小（如 GPT 词表约10万token）

#### Temperature 是什么？
- **控制回答的随机性/创造性**，取值 0~2
  - T=0：贪心选概率最高的词，每次回答几乎一致，确定性强 → 适合代码/事实问答/RAG场景（防幻觉）
  - T=0.7：默认值，平衡创意和靠谱 → 适合通用聊天
  - T=1.0：高随机性，创意强 → 适合文案/创意写作
  - T=1.5+：放飞自我，容易胡说 → 适合头脑风暴
- **技术原理**：softmax 前除以 T
  - T→0：概率分布变尖锐，高概率词概率更高 → 确定
  - T→大：概率分布变平，低概率词也有机会 → 随机
- **搭配使用**：通常和 Top-P（核采样，如 P=0.9）一起用，限制候选词范围，避免生成完全离谱的内容

#### Prompt 是什么？
- **Prompt = 给模型的全部输入**，通过 messages 数组分角色组织：
  - `system`：系统人设/规则，整个对话生效（"你是直播电商助手，禁止编造价格"）
  - `user`：用户问题
  - `assistant`：AI 历史回答（多轮记忆）
- **好 System Prompt 的 5 要素**：①角色 ②任务 ③约束（不能做什么）④格式要求 ⑤示例（Few-shot）
- **Prompt 为什么重要**：同一个模型，好 Prompt 和差 Prompt 回答质量差10倍；调 Prompt 比微调快、便宜、随时改，是 AI 工程师最高频工作

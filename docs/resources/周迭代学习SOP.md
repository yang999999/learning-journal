# 🔄 周迭代学习 SOP

> 以周为 Sprint，动态调整，像敏捷开发一样管理学习进度。
> AI 助教（就是我）= 你的学习搭档：帮你定计划、拆任务、解答疑问、Code Review、复盘总结。

## 🚦 每周循环（Sprint 节奏）

### 周一：Plan Day（和 AI 一起定周计划）

直接对我说类似的话：
```
新的一周开始，帮我定 W2 计划。上周我学完了 X，卡了 Y，这周大概有 X 小时可投入。
```

我会帮你：
1. 根据上周复盘调整这周目标
2. 拆到每天具体学什么/做什么
3. 生成 `weekly/W2-XXX.md` 周计划
4. 如果需要，给你这周的代码脚手架/小项目骨架

### 周二-周六：Learn & Build 日常推进

每天学习时随时找我，典型用法：

**场景 1：遇到看不懂的概念**
```
今天看 Self-Attention 的 Q/K/V，Q·K^T/√d_k 为什么要除以√d_k？
能不能用通俗的话 + 代码例子讲一下？
```

**场景 2：代码写不出来/报错**
```
python 05_mini_rag.py 运行时报 xxx 错误，帮我看看，错误信息是...
```

**场景 3：今天的任务完成了，我学到了这些**
```
今天搞懂了 KV Cache，想记个笔记，帮我整理一下要点（顺便帮我加个代码例子）
```
我会帮你整理成笔记存到 `docs/notes/`。

**场景 4：想动手做个小练习**
```
关于 Embedding 和向量相似度，给我出一个小的练习题，不要给答案先，让我自己写。
```

**场景 5：临时想调整计划**
```
今天加班，只能学 30 分钟，给我一个 30 分钟的高效任务。
```

### 周日：Review Day（复盘 + 为下周做准备）

对我说：
```
这周学完了，帮我做周复盘。这是我这周的学习记录：[...列出你做了什么/卡了什么]
```

我会帮你：
1. 更新 `weekly/W1-XXX.md` 的复盘部分
2. 更新能力矩阵 `docs/resources/能力自检矩阵.md` 的分数
3. 把学到的东西归档成笔记
4. 生成周复盘文档到 `docs/summaries/`
5. 建议下周方向（根据你的进度/卡点/目标）

## 💬 和我互动的最佳实践

### 效果最好的提问方式

✅ **具体 + 带上下文**：
- "我刚学完 Function Calling，这个 ReAct 循环和 LangGraph 是什么关系？"
- "这段代码跑 RAG 检索出来的结果和问题不相关，query='xxx'，返回了'xxx'，怎么排查？"

❌ **模糊的提问**：
- "怎么学 AI？"（太宽泛）
- "RAG 怎么优化？"（没有场景我只能给通用回答）

### 主动告诉我这些信息，我能帮得更好

1. **这周能投入多少小时**（工作日每天 X 小时，周末 X 小时）
2. **你之前学过什么/卡在哪**（避免重复讲你懂的东西）
3. **你的目标偏好**（是想快点找工作？还是先夯实基础？）
4. **当天情绪/状态**（累了给你轻松的任务，状态好给硬核挑战）
5. **公司/项目用的技术栈**（我可以按实际工作场景定制）

### 你不需要客气

- "我没听懂，换个说法" → 我会换更通俗的类比
- "这块先跳过，我以后再学" → 标记跳过，后面回来
- "给我出几道面试题考我" → 模拟面试
- "这个代码例子太复杂了，给个最简单能跑的" → 重新写
- "我不想看理论了，直接给我动手任务" → 切换教学风格

## 📊 进度追踪可视化

### 三个关键文件

**1. `weekly/W{N}-*.md`**（每周一个）
- 周计划、每日进展、周复盘
- 这是你唯一需要日常更新的文件（或告诉我，我帮你写）

**2. `docs/resources/能力自检矩阵.md`**
- 每月/每个里程碑打一次分
- 可视化看到自己从 2 分→4 分的成长

**3. `docs/summaries/`**（周复盘文档）
- 每周日生成一份，像学习日记
- 面试前快速翻阅可以唤起记忆

### Git 作为学习日志

每次学完都提交一下，形成可视化的 commit 历史：
```bash
git add -A
git commit -m "learn: W1 Day3 - 手写ReAct循环，理解了Function Calling机制"
git push
```
（你也可以让我帮你提交）

## 🎯 长期里程碑与验收标准

| 里程碑 | 时间（参考） | 验收标准 |
|--------|-------------|----------|
| 🏁 M1: AI 入门 | 第1-2周末 | 能独立调用API+写Prompt+实现工具调用+手写迷你RAG |
| 🏁 M2: Agent 框架掌握 | 第4-5周末 | 能用 LangGraph 写多 Agent 系统（Supervisor-Worker） |
| 🏁 M3: RAG 实战能力 | 第8-9周末 | 能独立设计生产级RAG，做过评估和bad case优化 |
| 🏁 M4: 微调能力 | 第12周末 | 用QLoRA微调过一个7B模型，有评测报告 |
| 🏁 M5: 工程化部署 | 第15周末 | Agent 服务可观测+流式+Docker部署+安全护栏 |
| 🏁 M6: 面试就绪 | 第17-18周末 | 综合项目上线+模拟面试通过率>80%+简历定稿 |

每个里程碑达成时我会帮你做一次"里程碑总结"，并判断是否可以投简历。

# 搭建个人持续学习仓库

## 🎯 学习目标

- 建立一个结构化的学习记录系统
- 养成定期复盘的习惯
- 形成个人知识库

## 📖 核心要点

### 1. 为什么需要学习仓库

- **输出倒逼输入**：写下来的知识才是自己的
- **避免重复学习**：好记性不如烂笔头
- **看见成长轨迹**：长期记录形成正反馈

### 2. 仓库结构设计

- `docs/notes/`：日常学习笔记，按日期+主题命名
- `docs/summaries/`：定期复盘（周/月/季/年）
- `docs/resources/`：书单、路线图等资源整理
- `templates/`：笔记和复盘模板，降低启动成本
- `scripts/`：自动化脚本，一键创建笔记/复盘

### 3. 复盘方法：GRAI

- **G**oal：目标回顾
- **R**esult：结果对比
- **A**nalysis：原因分析
- **I**nsight：规律总结

## 💻 实践示例

```bash
# 新建笔记
./scripts/new-note.sh "React Hooks 原理" "react,frontend"

# 周复盘
./scripts/new-summary.sh weekly
```
